# 変更履歴 (CHANGELOG)

## [0.56.71] - 2026-05-03

### 修正
- Gemini 3 思考モードが本文HTMLを書かずCTA素材だけを返してしまう問題への根本対応
  - YAML 形式のプロンプトを完全に廃止し、自然言語＋HTMLテンプレート例の構成に刷新（YAMLが「設定情報」と解釈されて instruction が後回しにされる挙動を回避）
  - 入力情報を `__INPUT_BLOCK__` プレースホルダーで上部に置き、出力フォーマット例とその下に「絶対ルール」を配置（最後に読まれる指示を Gemini 3 が重視する性質を活用）
  - 絶対ルールを「失敗扱いになる条件」として箇条書きで明示
  - 末尾に「応答は `<h1>` から始めてください」と最終命令を追加（recency 効果）
  - 不要になった YAML escape (`esc()`) 呼び出しを削除

---

## [0.56.70] - 2026-05-03

### 修正
- パネル表示用の内部バージョン定数 `CURRENT_VERSION` が `0.56.68` のままだったため、Tampermonkey上は更新されてもパネル右上の表示と起動ログが古いままになっていた問題を修正
- 起動ログを `CURRENT_VERSION` を参照するテンプレートリテラルに変更し、今後同じ取り違えが起きないよう一元化

---

## [0.56.69] - 2026-05-03

### 修正
- Gemini 3 思考モードがCTA素材ブロックだけを返してしまう問題に対応
  - MASTER_YAML 冒頭に「出力の絶対条件」コメントブロックを追加（最初の文字を `<h1>` にする等を明示）
  - 「2. 構成」を抽象的な日本語ラベルから、具体的なHTMLテンプレートに書き換え（CTA テンプレートと同じ粒度に揃える）
  - CTA テンプレートのプレースホルダ `[...]` を `（ここに〜を書く）` 形式に変更し、出力例ではなく明らかなダミーであることを示す
  - `version: "5.0-minimal-eisai"` から `minimal` を削除（モデルが「最小限の出力」と誤解しないように）
  - 本文HTMLの最低分量を明記（<h2>3個以上 / <p>8個以上 / 900字以上）
- バージョンを0.56.69に更新

## 開発方針

### バージョン管理ルール
- **コード変更時**: 必ずバージョンをインクリメント
- **同期**: blog-tools版とTampermonkey版で同一バージョンを維持
- **Git管理**: バージョン更新と同時にコミット・プッシュ

### 変更の種類
- **追加**: 新機能、新規記事タイプ
- **修正**: 既存機能の改善、バグ修正
- **削除**: 機能の削除（慎重に実施）

### リリースプロセス
1. コードの変更
2. バージョン番号の更新
3. 動作テスト
4. Gitコミット
5. GitHubプッシュ
6. Tampermonkey版の同期
