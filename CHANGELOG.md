## [Editor 0.9.0] - 2026-05-03

### 追加
- **表題フォーム（playground 内・ツールバー直下）**：エディタ本文の上に大型タイトル入力欄を新設。`position: sticky` でページスクロール時も常時上部に固定。
- **表題自動抽出**：「テーマを適用」実行時、GEMINI HTML の `<h1>` を表題フォームへ自動入力し、その要素は `tempDiv` から `removeChild` して**本文には含めない**。フォールバックは `<h2>` → `<h3>` → 本文先頭60字。
- **「📋 タイトルをコピー」ボタン**：表題フォーム横に配置。`<h1>表題</h1>` 形式でクリップボードへ。
- **「📋 本文をコピー」ボタン**：表題フォーム横に配置。テーマクラスをインラインstyleへ変換 + ブログCMS貼付用に整形 + 拡張機能ゴミ除去（後述）した本文HTMLをコピー。
- **拡張機能ゴミの自動除去 (`stripExtensionInjectedNodes`)**：LanguageTool（`.lg-badge-container`、`lt-*` 系）、Grammarly（`grammarly-*`、`[data-gramm*]`）等が contentEditable に注入するラッパー要素を、コピー時・保存時に自動でアンラップ／削除。CMS側で本文が表示されない問題に対応。
- **Gemini ライク履歴サイドバー**：
  - サイドバーを `position: sticky` 化し、ページスクロール時も常時表示
  - 履歴を**日付グループ表示**（今日 / 昨日 / 今週・過去7日 / 今月・過去30日 / それ以前）
  - 編集中エントリは黒地白字＋✏️＋「編集中」バッジ、その他は📌
  - ホバーで × 削除ボタン表示（Gemini と同じ操作感）
- **手動保存方式**（自動保存は無し）：
  - 「💾 履歴に残す」を押した時のみ保存
  - `Drafts.saveOrUpdate()`：現在編集中ID（インメモリ）が有効なら**同IDを更新**、無ければ新規作成
  - 同じ記事を続けて保存しても**新規履歴は作らず、既存履歴を更新**
  - ページ起動・リロード時はエディタを**完全まっさら**にし、過去編集の自動復元はしない（履歴サイドバーから明示選択する設計）
  - 「🆕 新規作成」は確認なしで即実行（前の編集は履歴に残るので失われない）
  - 編集中の状態は「● 未保存の変更あり」とステータス表示（書き戻しは無し）
- **ID重複自動排除 (`Drafts._dedupe`)**：load / save 時に同IDの重複を検出し、`updatedAt` が新しい方を残す。起動時に既存データを1回クリーンアップして再保存（過去バグの巻き添え対策）。
- **エディタ枠ごと保存された旧バグ履歴の自動修復 (`extractBodyHtml`)**：履歴クリック時に `.eisai-editor` 配下のみを抽出し、クリーン版で再保存。
- **JSON エクスポート / インポート**：履歴を JSON ファイルで端末間移行できる手動同期手段を提供。
- **レイアウト方針の刷新**：ページ全体スクロール方式へ変更（旧：エディタ内で内側スクロール）。本文が内容に応じて下方向に自然に伸びる。サイドバー・表題バーは sticky で常時可視。
- **`createEl` の id 属性バグ修正**：`Object.assign(el.style, { id: ... })` で id がスタイル扱いされ DOM 属性として設定されていなかった問題を修正。

### 補足
- 履歴は `localStorage` キー `eisai_editor_drafts_v2` に保存。容量目安は 5〜10MB（ブラウザ依存）／約 50〜250 件。`DRAFTS_MAX = 50`。
- 「現在編集中ID」(`_currentDraftId`) は**インメモリのみ**で保持し、リロードでリセットされる。
- 履歴は**ブラウザ別・端末別**。クロスデバイス同期は JSON エクスポート／インポートで手動対応（将来的に Supabase Storage 等での同期は別フェーズ）。
- ローカル開発用に `dev-server.py`（no-cache HTTP サーバ、ポート 8090）を追加。
- `reference/` に英才公式ブログのサンプル HTML / CSS（`default.css` / `common.css` / `fonts.css` / `category.css`）を取得済み（次フェーズの実プレビュー機能用）。
- バージョンを Ver.0.9.0 に更新（HTMLコメント・`<title>` の両方）。

## 開発方針

### バージョン管理ルール
- **コード変更時**: 必ずバージョンをインクリメント
- **同期**: blog-tools版とTampermonkey版で同一バージョンを維持
- **Git管理**: バージョン更新と同時にコミット・プッシュ

### 変更の種類
- **追加**: 新機能、新規記事タイプ
- **修正**: 既存機能の改善、バグ修正
- **削除**: 機能の削除（慎重に実施）

### リリースプロセス
1. コードの変更
2. バージョン番号の更新
3. 動作テスト
4. Gitコミット
5. GitHubプッシュ
6. Tampermonkey版の同期
